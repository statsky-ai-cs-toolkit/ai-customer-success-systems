# Signals Used for Churn Detection

These signals are designed to identify behavioral change, not just static health.

## Product Usage
- login frequency trend
- feature adoption (core features vs non-core)
- usage drop over time

## Engagement
- email engagement
- response rates
- meeting frequency

## Support Activity
- number of tickets
- severity of issues
- unresolved tickets

## Account Changes
- number of active users
- changes in usage patterns
- expansion or contraction signals

## Time-Based Signals
- time since last meaningful activity
- time to value achieved or not

---

## Key Principle

We are not looking for absolute values.

We are looking for **change over time**.

That is where risk shows up first.
